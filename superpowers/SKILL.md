# Superpowers — 思考方法论与工作流纪律

> 从 D:\MyAgents 的 Cursor skills 体系中提炼的核心理念，适配 OpenClaw agent 工作方式。
> 不是角色扮演，是可执行的分析框架。

## 什么时候用 / 不用

**用：**
- 深度分析问题、审视决策、方案评审
- 结构化表达（汇报、提案、复杂问题拆解）
- 认知蒸馏（从素材提炼思维模型）
- 设计自检（游戏方案、Boss 方案、Feature WHAT）

**不用：**
- 简单信息查询、文件操作、git 操作
- 纯技术排查（有明确报错信息时）

## 工作流（Agentic Protocol）

### Step 1: 问题分类 → 选择框架

| 信号 | 框架 | 核心方法 |
|------|------|---------|
| 战略/方向/资源分配 | **532 + 因果链** | 先问「对公式哪项有贡献」，再看分布再追因果 |
| 方案/设计评审 | **设计审核** | 6引擎检查 + 直觉层决策问题 |
| Boss 加分表演 | **Boss 预审** | 概念门→结构门→制作门 |
| Feature 设计 | **WHAT 审查** | 情绪目标→做对做坏→玩家视角→决策口 |
| 数据/指标异常 | **因果链穿透** | 先分布→再归因→反事实 |
| 管理/协作/组织 | **机制设计** | 四层职权 + 标尺共创 + 机制替代监工 |
| 表达/汇报 | **结构化表达** | 先结论→受众校准→有效信息公式 |
| 认知蒸馏 | **炼化炉** | 六维采集→框架提炼→Skill 构建 |
| 概念/想透 | **深度思维** | 解剖/追本/降秩 三选一 |

### Step 2: 执行框架

选定框架后，按该框架的纪律执行，不是罗列知识点。

#### 532 + 因果链（战略/资源）

1. **先问对公式哪项有贡献**——增长效能 = 单位成本产出的用户价值
2. **看结构再看绝对值**——先拆分布（头部/长尾/人群），再看总数
3. **穿透因果链**——指标变化归因到具体动作，不归因巧合
4. **反事实检验**——如果不做这个，会怎样？
5. **蠢指数审计**——系统开发维护总成本 / 核心体验绝对价值

#### 设计审核（方案评审）

6 引擎逐个扫，每引擎出 1-3 个发现：

| 引擎 | 问什么 |
|------|--------|
| 目的 | 目的清晰一致吗？服务谁？ |
| 节奏 | 情绪曲线有波峰波谷吗？ |
| 红线 | 触碰红线了吗？（核心循环不让步/不强迫/不冗余） |
| 价值 | 价值排序被违反了吗？（公平感>社交>爽感>留存>付费） |
| 分层 | 中坚层被保护了吗？ |
| 经济 | 通胀风险/追赶机制？ |

每轮 ≤3 个决策问题（Gate/Flag/Sense），人做最终判断。

#### 结构化表达（汇报/沟通）

**有效信息公式：谁 + 时间 + 事 + 程度**

1. 先锁受众——决策者看风险与动作，执行层看进度与阻塞
2. 先结论后展开——第一句话给判断，不铺垫
3. 说人话——先说「对你意味着什么」，再补技术细节
4. 密度控制——一段一观点，数据不超 4 组并排
5. 禁止流水账——读者不关心每人做了什么，关心哪里卡住了需要决策

#### 认知蒸馏（炼化炉）

六阶段流水线：

1. **入口分流**——自己人/外部人物/主题
2. **六维采集**——著作/对话/表达DNA/他者视角/决策记录/时间线
3. **框架提炼**——心智模型(3-7个，三重验证) + 启发式(5-10条) + 表达DNA
4. **Skill 构建**——Agentic Protocol 模板组装
5. **质量验证**——已知场景还原率 ≥70%
6. **精炼**——覆盖度 + 用户视角优化

### Step 3: 输出纪律

1. **先结论后论据**——不铺垫、不「自古以来」
2. **给选择不给指令**——提供方案让对方选
3. **能量化就量化**——不接受「感觉好了」
4. **痛点→解法，不上价值**——批评要跟解法
5. **列表优于段落**——结构优于修辞

## 算法五步法（顺序纪律）

任何优化/改进建议，必须按以下顺序检验：

1. **质疑需求**——这个需求该不该做？
2. **删除**——不产生价值的先砍掉
3. **简化**——剩余的能不能更简单？
4. **加速**——路径能不能更短？
5. **自动化**——最后才考虑 AI/自动化

**绝不能先自动化再问该不该存在。** 删除测试：如果删掉的 10% 不需要加回来，说明删得不够狠。

## 诚实边界

- 认知蒸馏产的是静态快照，不包含即时判断；涉及人事/组织决策需与本人确认
- 思维框架来自存量博弈期 MMO 经验，跨品类时部分启发式可能不适用
- 表达 DNA 从书面材料提炼，不代表口头沟通的全部风格
- 不替代人类直觉判断——AI 做逻辑完整性检查（快+全），人做直觉判断（不可替代）
